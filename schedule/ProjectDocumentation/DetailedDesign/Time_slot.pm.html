NAME

Time_slot - a time slot (day, start, duration)

VERSION

Version 1.00

SYNOPSIS

    use Schedule::Time_slot;
    
    my $time_slot = Time_slot->new (-day=>"Wed",
                                    -start=>"9:30",
                                    -duration=>1.5,
                                    -movable=>1);

DESCRIPTION

A time slot is specified by a day of the week, start time, length (in hours), and a whether or not it is allowed to move.

Example: the 'Block' object has a time slot used for teaching, whilst a 'Lab' object has a time slot indicating when it is not available.

METHODS

new ()

creates and returns a time_slot object

Parameters

-day => 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun'

-start => start time using 24 h clock (i.e 1pm is "13:00")

-duration => how long does this class last, in hours

Returns

Time_slot object

id ()

Returns the unique id for this time_slot object

day ( [day] )

Get/set the day of the week for this time_slot

start ( [time] )

Get/set start time of time_slot, in 24hr clock

end ( [time] )

Gets the end time in 24 hour clock

duration ( [duration] )

Gets and sets the length of the time_slot, in hours

movable ( [true | false] )

Gets and sets the course section object which contains this time_slot

start_number

Sets or returns the start time in hours (i.e. 1:30 pm = 13.5 hours)

This time info is set every time the start method is invoked on the object. Modifying this hash directly does not modify the value stored in 'start'.

To set the time according to this data (to the nearest grid level), use the method snap_to_time

day_number

Returns a hash ref that defines the day of the week as a real number, starting from monday (i.e. tues = 2.0)

This info is set every time the method is invoked on the object. Modifying directly does not modify the values stored in 'day'.

To set the day according to the data in this hash, use the method snap_to_day

snap_to_time

Takes the start_number, and converts it to the nearest fraction of an hour (if $Max_hour_div = 2, then snaps to every 1/2 hour).

Resets the 'start' property to the new clock time

Returns true if the new time is different than the previous time

snap_to_day

Takes the start_day, and converts it to the nearest day

Resets the 'day' property to the appropriate string

Returns true if the new time is different than the previous time

conflicts

Test that the current Time_slot conflicts with another time slot.
